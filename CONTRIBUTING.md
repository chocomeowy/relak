# Contributing to Relak

Thank you for helping us make Relak even better! This project is a group effort, and we want to ensure all changes are reviewed and approved by the team.

## Workflow

1.  **Work on a Branch**: Always create a new branch for your changes (e.g., `feature/ui-refresh`).
2.  **Make Changes**: Implement your updates and verify them locally.
3.  **Submit a Pull Request (PR)**:
    -   Push your branch and open a PR against the `main` branch.
    -   Provide a clear description of what changed and why.
4.  **Request Approval**:
    -   At least one other team member must review and **Approve** your PR before it can be merged.
    -   Once approved, you or the reviewer can merge the PR.

## Design Guidelines

-   **Zen & Minimal**: Keep the UI clean and focused on mental wellness.
-   **Smooth Animations**: Use `framer-motion` for transitions.
-   **Consistent Theme**: Follow the central design system (colors and typography) defined in `src/index.css`.

## Development

-   `npm run dev`: Launch the development server.
-   `npm run test`: Run the test suite.
-   `npm run build`: Create a production build.
